# CLAUSE 4 – CONTEXT OF THE ORGANIZATION

## Objective
To define the organization's context, interested parties, and the scope of the Information Security Management System (ISMS) for **CyTech Secure Solutions Ltd.**

## Implementation Summary
This section of the ISMS implementation focuses on identifying internal and external issues affecting CyTech Secure Solutions Ltd., defining relevant stakeholders, and establishing clear ISMS boundaries and scope.

### Planned Activities
1. Conduct **SWOT Analysis** to evaluate internal strengths, weaknesses, opportunities, and threats.
2. Conduct **PESTLE Analysis** to identify external factors influencing the ISMS.
3. Document all **interested parties** and their **information security expectations**.
4. Develop a formal **ISMS Context and Scope Statement** defining organizational and system boundaries.
5. Capture and store **evidence screenshots** of documentation and Git commits for audit verification.

### Deliverables
- ISMS_Context_and_Scope_Statement.docx  
- SWOT_Analysis.xlsx  
- PESTLE_Analysis.xlsx  
- Interested_Parties_and_Expectations.docx  
- ISMS_Boundary_Map.png  

### Evidence Collection
Screenshots of:
- Analysis spreadsheets  
- Folder structure  
- Document versions  
- Git commit logs  

### Folder Purpose
All files in this directory represent compliance activities under **Clause 4 – Context of the Organization** of **ISO/IEC 27001:2022**.
